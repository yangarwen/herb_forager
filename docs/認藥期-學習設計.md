# 用遊戲教會玩家認識中藥草 ——《本草堂》「認藥期」學習設計

> 用途：丟到對話中，以**學習設計／教學心理學**角度討論第一關「認藥期」怎麼設計得真正有效。
> 重點在「怎麼學得會、記得牢、用得出來」，技術實作其次。

## 背景（簡）
- 遊戲是古樸療癒的中藥房：玩家在第一人稱視角走動。核心玩法「配藥」是——客人帶**真實名方**上門，玩家照方到藥罐抓藥、交方。
- 共 100 味藥草，各有：中文名、性味（溫熱/平和/寒涼）、植物圖（古典彩繪）、一句白話功效。
- 想在配藥關之前加「認藥期」，讓玩家先熟悉藥草，再把所學用在配藥上。

## 學習目標
1. **圖 → 名（視覺辨識）**：看到藥草能說出名字。
2. **名/症 → 功效（語意記憶與應用）**：知道某味藥能做什麼、什麼症狀該用什麼藥。

這兩層是不同認知層次，打算分開、依序訓練（先辨識、後語意），避免認知過載。

---

## 「認藥期」設計草案（待評估與改進）

### 學習單元：以「批次」為單位（每批 8 味）
1. **【觀摩】** 看 8 張藥草圖 + 名稱 + 一句功效；可自由前後翻。旁白：「祖父拿出這味藥說……」
2. **【第一層測驗】圖 → 名（四選一）**：8 題，答錯立即顯示正解，不強制重來。
3. **【第二層測驗】症狀情境 → 選藥名（四選一）**：「有位老主顧常氣短乏力、說話沒力氣……」→ 選哪味藥。用情境包裝，不用「＿＿的功效是」。
4. **【批次小結】** 標出「已熟悉 ✓ / 需複習 ↺」；下次進遊戲先複習「需複習」，再給新的一批。

### 精熟與複習機制
- 每味藥有 `nameScore`、`effectScore`，各 0–3，到 3 算「熟悉」。
- 熟悉的藥不再進新批次，只偶爾複習（間隔複習，每味有 `lastReviewed` / `nextReview`）。

### 解鎖（學習 → 遊戲的正向循環）
按性味分區解鎖配藥關，而非學完 100 味才玩：
- 溫熱區 20 味 → 熟悉 16 味（80%）→ 解鎖左側藥罐
- 平和區 20 味 → 熟悉 16 味 → 解鎖後側藥罐
- 寒涼區 20 味 → 熟悉 16 味 → 解鎖右側藥罐

配藥關初期只出現以溫熱藥為主的方劑（理中丸、四逆湯、當歸補血湯…），學會的馬上能用。

## 想討論的問題（以學習成效為核心）
1. **認知層次與目標拆解**：分「辨識／語意應用」兩層合理嗎？是否該再細分（再認 → 回憶 → 應用）？順序與分關方式怎樣最利於記憶？
2. **批次大小與認知負荷**：一批 8 味是否恰當？要不要先小批（4–5）再加大？觀摩階段怎麼切分才不超載（分段、雙碼圖文）？
3. **檢索練習（testing effect）**：測驗作為「主動回憶」而非重看，怎麼安排最有效？要不要「先猜再看答案」、要不要無提示回憶（不只四選一）？
4. **間隔複習排程**：`nextReview` 用什麼間隔策略（固定？擴張間隔？簡化版 SM-2？）；答對/答錯如何調整 `score` 與下次間隔；如何兼顧「新藥」與「複習」的比例。
5. **精熟標準**：到 3 分算熟悉是否足夠？要不要區分「短期答對」與「隔日仍記得」？怎麼判定真的記住而非當下猜中。
6. **誘答（distractor）設計**：四選一的錯誤選項怎麼挑才有鑑別度、又能促進辨析（同性味？外觀相似？功效相近？），同時降低瞎猜。
7. **情境題與遷移**：第二層用「症狀情境」包裝，如何確保學到的能遷移到配藥關的真實任務？配藥關本身可否視為「應用層測驗」回饋學習資料？
8. **動機與情感**：無計時無失敗、敘事包裝、分區解鎖——還能怎麼維持投入又不變成枯燥背誦？

## ⚠️ 特別擔心的學習效度問題
藥草圖目前是**古典植物彩繪（整株植物）**，但玩家在配藥關、乃至現實中看到的是**乾燥飲片**。這樣會不會讓玩家「學會認植物、卻認不得藥材」？要不要圖文並陳（植物圖＋飲片照）、或分階段教？

## 技術備註（非重點）
純前端網頁（HTML/CSS/原生 JS + three.js，無建置）；玩家學習進度規劃用 Firebase Firestore 雲端存檔（每位玩家一份文件，記錄每味藥的 `seen` / `nameScore` / `effectScore` / `nextReview`、已解鎖性味區、目前批次）。等學習設計談定後再處理。

### Firestore 資料結構草案
```json
{
  "uid": "abc123",
  "nickname": "小藥童",
  "updatedAt": "2026-06-26T...",
  "herbs": {
    "當歸": {
      "seen": true,
      "nameScore": 2,
      "effectScore": 1,
      "lastReviewed": "2026-06-25",
      "nextReview": "2026-06-27"
    },
    "黃耆": { }
  },
  "unlockedRegions": ["溫熱"],
  "currentBatch": ["當歸", "黃耆", "川芎", "..."]
}
```

`nameScore` 與 `effectScore` 各 0–3，到 3 算「熟悉」。熟悉的藥不再出現在新批次、只偶爾複習。
